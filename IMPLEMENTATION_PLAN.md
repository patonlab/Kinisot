# Kinisot Remediation Plan

Step-by-step plan addressing the findings in [AUDIT.md](AUDIT.md). Ordered so
that each phase leaves the repo in a working, releasable state; later phases
build on earlier ones. Phases 1-3 are the high-value core; 4-6 are
modernization and features.

## Phase 0 — Safety net (do first, before touching behavior)

**Goal:** be able to detect any unintended numerical change.

1. Add characterization tests that pin down *current* behavior using the
   bundled examples, before any refactor:
   - Claisen KIE (`claisen_gs.out` / `claisen_ts.out`, labels from
     `claisen_kinisot.sh`), tetramethylcyclohexane EQE, and the existing
     Diels-Alder case at 2-3 temperatures and scaling factors.
   - Golden values generated by running the current code once.
2. Convert float assertions to `pytest.approx(..., rel=1e-6)`.
3. Set up GitHub Actions now (it gates everything after):
   `.github/workflows/ci.yml` running `pytest` on Python 3.9-3.13,
   ubuntu/macos/windows. Delete `.travis.yml`, `.travis-install.sh`, and the
   Travis badge (replace with an Actions badge).

**Exit criteria:** CI green on all platforms with ≥5 characterization tests.

## Phase 1 — Confirmed bug fixes (patch release, v2.0.3)

Each fix is one commit with a test that fails before and passes after.

1. **Constants block** (audit 1.2/1.3): delete the second constants block in
   `Kinisot.py`; keep the first (CODATA-correct) block. Update golden test
   values — this is a deliberate, documented numerical change of ~2×10⁻⁵ in
   frequencies (KIE changes will be far smaller).
2. **`NameError` in `compute_isotope_effect`** (audit 1.1): raise a
   `ValueError("transition structure has no imaginary frequency > cutoff")`
   from the library; catch it in `main()` and report via the logger. Test with
   a ground-state file passed as `--ts`.
3. **Scaling-factor lookup** (audit 1.5): match exactly after normalizing the
   Gaussian `R`/`U` prefix and case; stop at first match; fall back to 1.0
   with the existing warning. Add tests: exact match, CAM-B3LYP must *not*
   match B3LYP, unknown level → 1.0.
4. **Scaling table row** (audit 1.6): correct the duplicated
   `M06/maug-cc-pVTZ` row to `M06-2X/maug-cc-pVTZ` (verify against the Truhlar
   database v3b2 before committing).
5. **`is_linear`** (audit 1.4): replace the "prolate symmetric top" heuristic —
   determine linearity from the rotational constants (one missing/zero
   constant ⇒ linear) or from a near-zero-eigenvalue count of the mass-weighted
   Hessian. Add a linear-molecule test fixture (e.g. CO₂ or HCN freq output)
   and a non-linear symmetric-top fixture (e.g. NH₃).
6. Release v2.0.3 to PyPI; bump the conda-forge recipe.

**Exit criteria:** all six fixes tested; CHANGELOG entry explaining the
numerical change from the constants fix.

## Phase 2 — Robustness (v2.1.0)

1. **Exceptions, not exits** (audit 2.2): in `Hess_to_Freq.py`, replace
   `sys.exit()` with a `KinisotParseError` exception (new `exceptions.py` or
   defined alongside). `main()` is the only place allowed to call `sys.exit`.
2. **Parse validation** (audit 2.1):
   - `read_hess`: explicit errors for missing `NAtoms=`, missing archive
     (`NImag`) section, mass-count/atom-count mismatch, and `--iso` indices
     out of range.
   - `level_of_theory`: return `None`/raise cleanly when no `\Freq\` archive
     line exists; handle the unbound `bs`.
   - Warn (don't `nan`) when >1 imaginary frequency survives the cutoff.
3. **Context managers** (audit 2.3): `with open(...)` everywhere; make
   `Logger` a context manager (`__enter__`/`__exit__`) so the `.dat` file is
   always closed; call it from `main()`.
4. **Imports** (audit 2.2): delete the try/except import shim and `import *`;
   use explicit relative imports (`from .Hess_to_Freq import read_hess,
   level_of_theory, is_linear`). `python -m kinisot` covers the script use
   case, so the non-package fallback can go.
5. **Numerical cleanups** (audit 2.4): replace `np.log(math.exp(x))` with `x`;
   delete dead `hess_mat` and the rounded-frequency dead code.
6. **CLI fixes** (audit 2.5): `-s` default `None` (not `False`);
   error out if both `--ts` and `--prd` are given; use `parse_args()` (reject
   unknown flags); add `--version`; document the `'0'` no-substitution label
   in `--help` and README; remove the unreachable `sys.exit()`.
7. Expand tests alongside each item: EQE path, truncated file, single-point
   file, bad `--iso` index, CLI smoke test via `python -m kinisot` on the
   examples (assert on the produced `.dat`).

**Exit criteria:** no `sys.exit`/bare `except`/unclosed file in library code;
error paths covered by tests; coverage reported in CI (target ≥80%).

## Phase 3 — Packaging & project modernization (v2.1.0, same release)

1. Add `pyproject.toml` (PEP 621) with all metadata; delete `setup.py` and
   `setup.cfg` (drop the `universal=1` py2 wheel flag). Single-source the
   version: `__version__` in `kinisot/__init__.py`, read by the build backend
   (e.g. hatchling or setuptools `attr:`), referenced by `recipe/meta.yaml`
   via the tarball.
2. Set `requires-python = ">=3.9"` (NumPy 2 floor) and declare
   `dependencies = ["numpy>=1.22"]`.
3. Add a console entry point: `kinisot = kinisot.Kinisot:main` so users can
   run `kinisot --rct ... --ts ...` directly.
4. Populate `kinisot/__init__.py`: `__version__` and re-export the public API
   (`compute_isotope_effect`, `calc_rpfr`, `read_hess`).
5. Remove Python-2 relics (`from __future__ import ...`), unused imports
   (`pathlib`, `glob`, `math` in `Hess_to_Freq`), and the pip-copied
   `__main__.py` boilerplate (reduce to `from .Kinisot import main; main()`).
6. Move `kinisot/examples/` → `tests/data/` (keep a small curated
   `examples/` at repo root for documentation); stop shipping multi-MB
   Gaussian outputs in the wheel. Update `MANIFEST.in`/`conftest.py`.
7. Add linting to CI: `ruff` (style + unused imports) and `ruff format` or
   `black`; fix the 3-space indentation as a one-time mechanical commit.
8. Release v2.1.0; update conda-forge recipe.

**Exit criteria:** `pip install .` from a clean checkout works via
`pyproject.toml` only; wheel < 100 kB; `kinisot --version` works; lint clean.

## Phase 4 — Internal refactor (v2.2.0)

**Goal:** separate parsing / physics / presentation so the code is testable
and extensible. Golden tests from Phase 0 guarantee numerical invariance.

1. Restructure modules:
   - `kinisot/parsers/gaussian.py` — file parsing (`read_hess`,
     `level_of_theory`, geometry/rotational constants).
   - `kinisot/thermo.py` — constants (CODATA 2018, cited) and the
     RPFR/Bigeleisen-Mayer math. Convert `calc_rpfr` from a
     class-used-as-function into a dataclass returned by a pure function.
   - `kinisot/cli.py` — argument parsing and all `Logger` formatting.
   - Keep `kinisot.Kinisot` as a thin deprecation shim for one minor release.
2. Make `compute_isotope_effect` return a structured result (dataclass with
   named fields) instead of an 8-tuple; CLI formats it.
3. Replace the NumPy structured array in `vib_scale_factors.py` with a plain
   dict `{level: ScaleFactor}` (drops the `bytes.decode()` noise and the
   numpy import).
4. Vectorize the three factor functions with NumPy (`np.log`, `np.expm1`)
   over frequency arrays — clearer and faster, and eliminates the
   per-element `math.exp` overflow risk at low temperature.

**Exit criteria:** golden tests unchanged to 1e-9 relative on KIE outputs
(except where Phase 1 already moved them); public API documented in README.

## Phase 5 — Feature gaps (v2.3.0, prioritize by user demand)

1. **Isotope table**: replace the three hard-coded `np.isclose` mass swaps
   with a proper table (¹H→²H/³H, ¹²C→¹³C/¹⁴C, ¹⁴N→¹⁵N, ¹⁶O→¹⁷O/¹⁸O, …) and a
   CLI syntax to choose the target isotope, e.g. `--iso 5:18O,7:2H`
   (keep the current numeric form as a synonym for the default heavy isotope).
2. Optional trans/rot mode projection (Gaussian-style) as `--project`,
   complementing the current low-mode-dropping approach.
3. Other QM codes (ORCA `.hess`, at least) via the Phase 4 parser interface —
   longstanding ecosystem ask; the parser split makes it a contained change.
4. `--output` path option and a `--quiet` flag for the logger; machine-readable
   output (`--json`).
5. Docs: expand README (EQE mode, multi-reactant labels, `'0'` convention,
   tunneling model caveats); add CITATION.cff pointing at the Zenodo DOI.

## Phase 6 — Release hygiene (ongoing)

1. CHANGELOG.md (Keep a Changelog format), retroactively covering 2.0.x.
2. Tag-driven publishing: GitHub Actions workflow to build sdist/wheel and
   publish to PyPI (trusted publishing) on tag push.
3. Enable Dependabot/renovate for the Actions workflow versions.

## Suggested sequencing at a glance

| Phase | Deliverable | Release | Risk |
|-------|-------------|---------|------|
| 0 | Characterization tests + GitHub Actions | — | none |
| 1 | Six confirmed-bug fixes | v2.0.3 | small documented numerical shift |
| 2 | Error handling, CLI fixes, test expansion | v2.1.0 | low |
| 3 | pyproject.toml, entry point, repo cleanup | v2.1.0 | low |
| 4 | parser/physics/CLI split, structured results | v2.2.0 | medium (mitigated by goldens) |
| 5 | isotope table, new parsers, output options | v2.3.0 | scoped per feature |
| 6 | changelog + automated publishing | ongoing | none |

Phases 0-1 are roughly a day of work; 2-3 a few days; 4 is the largest single
chunk and should be one reviewed PR per module move.
